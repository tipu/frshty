Change Summary

RPA execution logs now support optional screenshot attachments captured by the client and uploaded to S3, with Django generating deterministic presigned URLs (24-hour windows) for browser caching and the frontend displaying them lazily with automatic expiration fallback after 30 days. Non-blocking async uploads decouple image delivery from log creation, while infrastructure enforces S3-managed encryption, role-scoped write access by tenant/queue prefix, and automated lifecycle deletion.

Description

Create a new Logs S3 bucket for storing our log screenshots. We could consolidate our buckets so we don't need to manage so many. Whether this is a separate bucket, or a folder within a bucket, doesn't matter too much. But storing them in the "invoices" bucket would not make sense, so we would need to rename it at least, which would be a fair amount of work...
